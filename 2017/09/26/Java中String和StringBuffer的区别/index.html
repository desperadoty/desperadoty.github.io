Java中String和StringBuffer的区别

发表于 2017-09-26 | 分类于 Java
  • String是不可变对象，一旦被创建，值就不能修改，是线程安全的。String类被final修饰，所以不能被继承。
  • StringBuffer是可变对象。它对方法加了同步锁或者对调用的方法加了同步锁，所以也是线程安全的。
  • String重写了equals方法和hashCode方法
  • StringBuffer未重写equals方法和hashCode方法

2018美团笔试
String s = new String(“str”) + new String(“ing”)创建了几个String对象?

答案是5个String对象，1个StringBuilder对象

“str”和”ing”两个对象，在类加载阶段，JVM会在堆中创建这两个字符串对象实例，前提是这两个对象之前并没有出现过，然后常量池中会驻留这两个对象的引用，而对象本身并不在常量池中。

每出现一个new关键字，就会创建一个对象。所以又会有两个对象。

最后字符串相加的操作，由于String对象不可变，所以这里创建了一个新的”string”对象，s指向这个对象的引用。

以上，共有5个String对象。

接下来是StringBuilder对象出现在哪里？

原来，字符串相加的操作实际创建了一个StringBuilder对象，然后每次调用它的append()方法，实现字符串相连，最后调用一个toString()方法，在toString()方法中会创建一个新的String对象并返回。
