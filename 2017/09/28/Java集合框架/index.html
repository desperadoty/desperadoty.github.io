Java集合框架

Posted on 2017-09-28

集合类型主要分为Set(集), List(列表), Map(映射)。
集合类存放的是对象的引用，而非对象本身。

List

List接口继承Collection接口，允许集合中存在重复项。List中的元素是有序的。

常用的实现类：

  • ArrayList
    可变大小的数组。
  • LinkedList
    是一种可以在任何位置进行高效的插入和删除操作的有序序列。

Set

Set接口继承Collection接口，不允许集合中存在重复项。Set中的元素是无序的。

常用的实现类：

  • HashSet
    按照哈希算法存取集合中的对象，存取速度比较快。允许包含值为null的元素，但最多只能有一个null元素。采用散列的存储方法，所以没有顺序，元素的插入顺序和输出顺序不一致。
  • TreeSet
    实现了SortedSet接口，能够对集合中的对象进行排序。
  • LinkedHashSet
    保存了元素的添加顺序，所以其中元素是有序的。

Map

Map接口是一组成对的键-值对象，即所持有的是key-value pairs。Map中不允许有重复的key。

常用的实现类：

  • HashMap
    根据key的HashCode值存储数据，根据key可以直接获取它的value，具有很快的访问速度。最多只允许一条记录的key为null(多条会覆盖)，允许多条记录的value为null。非同步。
  • TreeMap
    其中键值对默认按照key升序排序。使用Iterator遍历时得到的结果是有序的。不允许key的值为null。非同步。
  • LinkedHashMap
    保存了记录的添加顺序，使用Iterator遍历时，先得到的记录肯定是先添加的。允许key和value为null。非同步。
  • Hashtable
    不允许key和value的值为null。同步。
