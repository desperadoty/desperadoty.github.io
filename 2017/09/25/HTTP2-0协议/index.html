HTTP2.0协议

Posted on 2017-09-25

基本概念

  1. HTTP(HyperText Transfer Protocol), 超文本传输协议。
  2. HTTPS(HyperText Transfer Protocol Secure), 超文本传输安全协议。
  3. SPDY, 一种开放的网络传输协议，由Google开发，用来传输网页内容。基于传输控制协议(TCP)的应用层协议。SPDY也就是HTTP/2的前身。

HTTP与HTTPS的区别

  1. HTTP协议运行在TCP之上，明文传输；HTTPS经由HTTP进行通讯，但利用SSL/TSL来加密封包。
  2. 二者连接方式不同，HTTP端口号为80，HTTPS端口号为443。

TCP与UDP的区别

TCP(Transmission Control Protocol), 传输控制协议是一种面向连接、可靠的、基于字节流的传输层通信协议。
UDP(User Datagram Protocol), 用户数据报协议是一个简单的面向数据报的传输层协议。

  1. TCP面向连接，UDP面向无连接。
  2. TCP提供可靠的服务，保证数据的正确性和数据顺序，UDP提供不可靠的服务，可能会导致数据缺失，也不保证数据顺序。
  3. TCP用于传输大量数据（流模式），UDP用于传输少量数据（数据报模式）。
  4. TCP对系统资源要求多，UDP相对较少。

HTTP2.0新特性

  1. Binary Format.HTTP1.x的协议解析是基于文本，HTTP2.0的协议解析采用的是二进制格式，实现方便且健壮。
  2. MultiPlexing.多路复用，即连接共享，每一个request都是用作连接共享机制的。一个request对应一个id，这样一个连接上可以有多个request，每个连接的request可以随机的混在一起，接收方可以根据request的id将request归属到各自不同的服务端请求里。该特性消除了同域名并发连接数限制。
  3. Header Compression.HTTP1.x的header带有大量信息，而且每次都要重复发送，HTTP2.0使用encoder来减少需要传输的header大小，通讯双方各自缓存一份header fields表，既避免了重复header的传输，又减少了需要传输的大小。
  4. Server Push.服务端推送。服务端向客户端发送比客户端请求更多的数据，这允许服务器直接提供浏览器渲染页面所需资源，而无需浏览器在收到、解析页面后再提起一轮请求，节约了加载时间。
