Java多线程

Posted on 2017-09-28

在OS中引入进程的目的是为了使多个程序能并发执行，以提高资源利用率和系统吞吐量，引入线程的目的是为了减少程序在并发执行时所付出的时空开销，使OS具有更好的并发性。

在引入线程的OS中，进程是拥有资源的基本单位，线程是调度和分派的基本单位。

多线程的实现方式

  • 继承Thread类


  • 实现Runnable接口


从上面的实例来看，直接继承Thread类，创建的3个线程每一个都售出3张票；而实现Runnable接口，创建的3个线程一起共售出3张票，更符合现实需求。

一般都选择实现Runnable接口，主要有如下优势：

  • 可以避免由于Java单继承带来的弊端
  • 可以增强程序的健壮性，代码能够被多个线程共享，代码与数据是独立的
  • 适合多个相同程序代码的线程区处理同一资源的情况

Java集合框架

Posted on 2017-09-28 | In Java

集合类型主要分为Set(集), List(列表), Map(映射)。
集合类存放的是对象的引用，而非对象本身。

List

List接口继承Collection接口，允许集合中存在重复项。List中的元素是有序的。

常用的实现类：

  • ArrayList
    可变大小的数组。
  • LinkedList
    是一种可以在任何位置进行高效的插入和删除操作的有序序列。

Set

Set接口继承Collection接口，不允许集合中存在重复项。Set中的元素是无序的。

常用的实现类：

  • HashSet
    按照哈希算法存取集合中的对象，存取速度比较快。允许包含值为null的元素，但最多只能有一个null元素。采用散列的存储方法，所以没有顺序，元素的插入顺序和输出顺序不一致。
  • TreeSet
    实现了SortedSet接口，能够对集合中的对象进行排序。
  • LinkedHashSet
    保存了元素的添加顺序，所以其中元素是有序的。

Map

Map接口是一组成对的键-值对象，即所持有的是key-value pairs。Map中不允许有重复的key。

常用的实现类：

  • HashMap
    根据key的HashCode值存储数据，根据key可以直接获取它的value，具有很快的访问速度。最多只允许一条记录的key为null(多条会覆盖)，允许多条记录的value为null。非同步。
  • TreeMap
    其中键值对默认按照key升序排序。使用Iterator遍历时得到的结果是有序的。不允许key的值为null。非同步。
  • LinkedHashMap
    保存了记录的添加顺序，使用Iterator遍历时，先得到的记录肯定是先添加的。允许key和value为null。非同步。
  • Hashtable
    不允许key和value的值为null。同步。

HTTP状态码及含义

Posted on 2017-09-28 | In NetWork

HTTP状态码(HTTP Status Code)是用以表示网页服务器(Web Server)超文本传输协议(HTTP)响应状态的3位数字代码。

1xx 消息，服务器成功收到请求，需要请求者继续执行操作

2xx 成功，操作被成功接收并处理

3xx 重定向，需要进一步的操作以完成请求

4xx 客户端错误，请求包含语法错误或无法完成请求

5xx 服务器错误，服务器在处理请求的过程中发生了错误

常见HTTP状态码如下：

200 OK 请求成功

301 Moved Permanently 资源被永久转移到其他URL

400 Bad Request 客户端请求的语法错误，服务器不能或不会处理该请求

401 Unauthorized 当前请求需要用户验证

403 Forbidden 服务器已经理解请求，但还是拒绝执行它

404 Not Found 请求失败，服务器无法根据客户端的请求找到资源

500 Internal Server Error 内部服务器错误

502 Bad Gateway 充当网关或代理的服务器，从远端服务器接收到了一个无效的请求

504 Gateway Time-out 充当网关或者代理的服务器，未及时从远端服务器获取请求

Java中String和StringBuffer的区别

Posted on 2017-09-26 | In Java
  • String是不可变对象，一旦被创建，值就不能修改，是线程安全的。String类被final修饰，所以不能被继承。
  • StringBuffer是可变对象。它对方法加了同步锁或者对调用的方法加了同步锁，所以也是线程安全的。
  • String重写了equals方法和hashCode方法
  • StringBuffer未重写equals方法和hashCode方法

2018美团笔试
String s = new String(“str”) + new String(“ing”)创建了几个String对象?

答案是5个String对象，1个StringBuilder对象

“str”和”ing”两个对象，在类加载阶段，JVM会在堆中创建这两个字符串对象实例，前提是这两个对象之前并没有出现过，然后常量池中会驻留这两个对象的引用，而对象本身并不在常量池中。

每出现一个new关键字，就会创建一个对象。所以又会有两个对象。

最后字符串相加的操作，由于String对象不可变，所以这里创建了一个新的”string”对象，s指向这个对象的引用。

以上，共有5个String对象。

接下来是StringBuilder对象出现在哪里？

原来，字符串相加的操作实际创建了一个StringBuilder对象，然后每次调用它的append()方法，实现字符串相连，最后调用一个toString()方法，在toString()方法中会创建一个新的String对象并返回。

当你输入一个网址的时候

Posted on 2017-09-26 | In NetWork

当你输入一个网址的时候，实际会发生什么？

  1. 在浏览器中输入网址，按下回车

  2. 浏览器通过DNS查找域名的IP地址

    DNS(Domain Name System)域名系统，因特网上作为域名和IP地址相互映射的一个分布式数据库，能够使用户更方便的访问互联网，而不用去记住能够被机器直接读取的IP数串。通过主机名，最终得到对应的IP地址的过程叫做域名解析。

  3. 浏览器给WEB服务器发送一个HTTP请求

    GET请求定义了要读取的URL，也包含浏览器存储的该域名的cookies。在不同页面的请求中，cookies是与跟踪一个网站状态相匹配的键值。这样cookies会存储登录用户名，服务器分配的密码和一些用户设置等。cookies会以文本文档形式存储在客户机里，每次请求时发送给服务器。

  4. 服务器给浏览器一个301永久重定向响应

    页面永久性移走（301重定向）是一种非常重要的“自动转向”技术，是网址重定向最为可行的一种办法。当用户或搜索引擎向网站服务器发出浏览请求时，服务器返回的HTTP数据流中header的状态码的一种，表示本网页永久性转移到另一个地址。简单地说，301重定向会把不带www的地址转移到带www的网站地址。

  5. 浏览器跟踪重定向地址

    浏览器知道正确的访问地址后，会向它发送一个新的GET请求。

  6. 服务器处理请求

  7. 服务器发回一个HTML响应

  8. 浏览器开始显示HTML

  9. 浏览器发送获取嵌入在HTML中的对象

  10. 浏览器发送异步（AJAX）请求

HTTP2.0协议

Posted on 2017-09-25 | In NetWork

基本概念

  • HTTP(HyperText Transfer Protocol), 超文本传输协议。
  • HTTPS(HyperText Transfer Protocol Secure), 超文本传输安全协议。
  • SPDY, 一种开放的网络传输协议，由Google开发，用来传输网页内容。基于传输控制协议(TCP)的应用层协议。SPDY也就是HTTP/2的前身。

HTTP与HTTPS的区别

  • HTTP协议运行在TCP之上，明文传输；HTTPS经由HTTP进行通讯，但利用SSL/TSL来加密封包。
  • 二者连接方式不同，HTTP端口号为80，HTTPS端口号为443。

TCP与UDP的区别

TCP(Transmission Control Protocol), 传输控制协议是一种面向连接、可靠的、基于字节流的传输层通信协议。
UDP(User Datagram Protocol), 用户数据报协议是一个简单的面向数据报的传输层协议。

  • TCP面向连接，UDP面向无连接。
  • TCP提供可靠的服务，保证数据的正确性和数据顺序，UDP提供不可靠的服务，可能会导致数据缺失，也不保证数据顺序。
  • TCP用于传输大量数据（流模式），UDP用于传输少量数据（数据报模式）。
  • TCP对系统资源要求多，UDP相对较少。

HTTP2.0新特性

  • Binary Format.HTTP1.x的协议解析是基于文本，HTTP2.0的协议解析采用的是二进制格式，实现方便且健壮。
  • MultiPlexing.多路复用，即连接共享，每一个request都是用作连接共享机制的。一个request对应一个id，这样一个连接上可以有多个request，每个连接的request可以随机的混在一起，接收方可以根据request的id将request归属到各自不同的服务端请求里。该特性消除了同域名并发连接数限制。
  • Header Compression.HTTP1.x的header带有大量信息，而且每次都要重复发送，HTTP2.0使用encoder来减少需要传输的header大小，通讯双方各自缓存一份header fields表，既避免了重复header的传输，又减少了需要传输的大小。
  • Server Push.服务端推送。服务端向客户端发送比客户端请求更多的数据，这允许服务器直接提供浏览器渲染页面所需资源，而无需浏览器在收到、解析页面后再提起一轮请求，节约了加载时间。
