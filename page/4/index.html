Java线程五大状态

Posted on 2017-10-12 | In Java

Java线程的五种状态：创建、就绪、运行、阻塞和死亡。

创建(New)

仅创建了线程对象，并没有调用其start()方法。

就绪(Runnable)

当线程的start()方法被调用后，线程进入就绪状态，等待获取CPU资源。

运行(Running)

当线程获取了CPU资源后，进入运行状态，执行run()方法中的程序代码。

阻塞(Blocked)

当正在运行的线程因某种原因暂时停止运行时，线程进入阻塞状态。
1.等待阻塞：
运行的线程执行某个对象的wait()方法，线程会被置入该对象的等待集(Wait Pool)中。
2.同步阻塞：
运行的线程在获取对象同步锁时，若同步锁被别的线程占用，线程会被置入锁定集(Lock Pool)中。
3.其他阻塞：
运行的线程执行sleep()或join()方法，或者发出了I/O请求，线程会被置入阻塞集(Block Pool)中。

死亡(Dead)

当线程的run()方法执行结束后进入死亡状态。此外，若线程执行了interrupt()或stop()方法，那么它也会以异常退出的方式进入死亡状态。

Shiro权限管理框架

Posted on 2017-10-09 | In Shiro

Apache Shiro是一个开源安全框架，提供身份验证、授权、密码学和会话管理。Shiro框架直观、易用，同时也能提供健壮的安全性。

Shiro基础

Subject: 主体

Realm: 验证主体的数据源

Authentication: 身份认证/登录，验证用户是不是拥有相应的身份
用户需要提供principals(身份)和credentials(证明)给Shiro,从而应用能够验证用户身份
一个主体可以有多个principals，但只有一个primary principals，一般是用户名/密码/手机号

Authorization: 授权，即权限验证，验证某个已认证的用户是否拥有某个权限

Session Manager: 会话管理，会话是用户访问应用时保持的连接关系，在多次交互中应用能够识别出当前访问的用户是谁，且可以在多次交互中保存一些数据。

隐式角色: 基于角色的访问控制

显示角色: 基于资源的访问控制

编码／解码: base64和16进制

散列算法一般用于生成数据的摘要信息，是一种不可逆的算法，一般适合存储密码之类的数据，常见散列算法如MD5、SHA等。一般进行散列时最好提供一个salt(盐)，如果直接对密码进行散列相对来说破解更容易，我们可以加一些只有系统知道的干扰数据，如用户名和ID(即盐)，这样散列的对象是“密码＋用户名＋ID”，这样生成的散列值相对来说更难破解。

Java23种设计模式之单例模式

Posted on 2017-10-09 | In Java

Java23种设计模式：
Abstract Factory(抽象工厂模式)、Builder(建造者模式)、Factory Method(工厂方法模式)、Prototype(原始模型模式)、Singleton(单例模式)、Facade(门面模式)、Adapter(适配器模式)、Bridge(桥梁模式)、Composite(合成模式)、Decorator(装饰模式)、Flyweight(享元模式)、Proxy(代理模式)、Command(命令模式)、Interpreter(解释器模式)、Visitor(访问者模式)、Iterator(迭代子模式)、Mediator(调停者模式)、Memento(备忘录模式)、Observer(观察者模式)、State(状态模式)、Strategy(策略模式)、Template Method(模板方法模式)、Chain Of Responsibility(责任链模式)。

单例模式：
确保一个类只有一个实例，而且自行实例化并向整个系统提供这个实例。

饿汉式单例

饿汉式单例在单例类被加载时就实例化一个对象。

懒汉式单例

懒汉式单例在调用getInstance方法时实例化对象。

单例模式的优点

  • 只有一个可全局访问的对象，节省内存空间。
  • 避免频繁地创建和销毁对象，可以提高性能。
  • 避免对共享资源的多重占用。

Nginx负载均衡与反向代理

Posted on 2017-10-07 | In NetWork

负载均衡

负载均衡(Loading Balancing)，是一种计算机网络技术，用来在多个计算机（计算机集群）、网络连接、CPU、磁盘驱动器或其他资源中分配负载，以达到最优化资源使用、最大化吞吐率、最小化响应时间、同时避免过载的目的。

反向代理

反向代理(Reverse Proxy)，是一种代理服务器。它根据客户端的请求，从后端的服务器（如WEB服务器）上获取资源，然后再将这些资源返回给客户端。与前向代理不同，前向代理作为一个媒介将互联网上获取的资源返回给相关联的客户端，而反向代理是在服务器端作为代理使用，而不是客户端。客户端通过前向代理可以访问很多不同的资源，而反向代理是很多客户端都通过它访问不同后端服务器上的资源，而不需要知道这些后端服务器的存在，而以为所有资源都来自愿这个反向代理服务器。

Nginx安装与配置

1.安装
brew install nginx
2.配置
sudo vim /usr/local/etc/nginx/nginx.conf

3.常用命令
启动nginx sudo nginx
重启nginx sudo nginx -s reload
停止nginx sudo nginx -s stop

Nginx反向代理

1.通过修改hosts文件模拟多个域名访问本机的nginx。
sudo vim /etc/hosts

2.修改nginx配置文件设置反向代理
sudo vim /usr/local/etc/nginx/nginx.conf

3.重启nginx服务
sudo nginx -s reload
此时，在浏览器访问testnginx.com会访问到www.bilibili.com;访问testreverse.com会访问到www.acfun.cn;访问testproxy.com会访问到www.zhihu.com。

MySQL重置密码

Posted on 2017-10-06 | In DataBase

MySQL安装时会有一个临时密码，如下图：

在知道临时密码的情况下，我直接使用
mysqladmin -u root password 'your new password'
然而，今天发现重启过电脑之后就不再生效了。

改用以下方法重置密码：
1.停止MySQL服务器
2.cd /usr/local/mysql/bin
3.sudo su
4.输入开机密码
5.sudo /usr/local/mysql/bin/mysqld_safe --skip-grant-tables
6.重新打开一个新的终端
7.sudo /usr/local/mysql/bin/mysql -u root
8.输入开机密码
9.UPDATE mysql.user SET authentication_string=PASSWORD('your new password' WHERE User='root';
10.FLUSH PRIVILEGES;
11.\q

修改密码后确实能够使用新密码登录，但可能会遇到如下错误：
You must reset your password using ALTER USER statement before executing this statement
解决方法如下：
1.mysql>SET PASSWORD = PASSWORD('your new password');
2.mysql>ALTER USER 'root'@'localhost' PASSWORD EXPIRE NEVER;
3.mysql>FLUSH PRIVILEGES;

mysqldump -u root -p test > /home/bak.sql //备份test数据库

cat bak.sql //查看备份文件是否与原来数据库一致

mysql -u root -p test < /home/bak.sql //执行备份文件进行恢复

MySQL数据库优化

Posted on 2017-10-03 | In DataBase

1.表的设计要规范。

2.适当建立索引，在频繁作为检索条件，更新较少的字段上建立索引，以提高查询速度。

3.分表查询，包括水平分割和垂直分割。

4.读写分离，读(read)，写(create、update、delete)。

5.建立存储过程。

数据库范式：1NF、2NF、3NF、BCNF、4NF、5NF（由高到低）。

规范化：一个低一级范式的关系模式通过模式分解可以转换为若干个高一级范式的关系模式的集合。

1NF: 每一个分量都是不可分割的数据项。

2NF: 不存在非主属性对码的部分函数依赖。

3NF: 不存在非主属性对码的传递函数依赖。

BCNF: 决定因素都是码。不存在主属性对码的部分和传递函数依赖。

4NF: 不存在多值依赖。
