Java23种设计模式之单例模式

Posted on 2017-10-09 | In Java

Java23种设计模式：
Abstract Factory(抽象工厂模式)、Builder(建造者模式)、Factory Method(工厂方法模式)、Prototype(原始模型模式)、Singleton(单例模式)、Facade(门面模式)、Adapter(适配器模式)、Bridge(桥梁模式)、Composite(合成模式)、Decorator(装饰模式)、Flyweight(享元模式)、Proxy(代理模式)、Command(命令模式)、Interpreter(解释器模式)、Visitor(访问者模式)、Iterator(迭代子模式)、Mediator(调停者模式)、Memento(备忘录模式)、Observer(观察者模式)、State(状态模式)、Strategy(策略模式)、Template Method(模板方法模式)、Chain Of Responsibility(责任链模式)。

单例模式：
确保一个类只有一个实例，而且自行实例化并向整个系统提供这个实例。

饿汉式单例

饿汉式单例在单例类被加载时就实例化一个对象。

懒汉式单例

懒汉式单例在调用getInstance方法时实例化对象。

单例模式的优点

  • 只有一个可全局访问的对象，节省内存空间。
  • 避免频繁地创建和销毁对象，可以提高性能。
  • 避免对共享资源的多重占用。

Nginx负载均衡与反向代理

Posted on 2017-10-07 | In NetWork

负载均衡

负载均衡(Loading Balancing)，是一种计算机网络技术，用来在多个计算机（计算机集群）、网络连接、CPU、磁盘驱动器或其他资源中分配负载，以达到最优化资源使用、最大化吞吐率、最小化响应时间、同时避免过载的目的。

反向代理

反向代理(Reverse Proxy)，是一种代理服务器。它根据客户端的请求，从后端的服务器（如WEB服务器）上获取资源，然后再将这些资源返回给客户端。与前向代理不同，前向代理作为一个媒介将互联网上获取的资源返回给相关联的客户端，而反向代理是在服务器端作为代理使用，而不是客户端。客户端通过前向代理可以访问很多不同的资源，而反向代理是很多客户端都通过它访问不同后端服务器上的资源，而不需要知道这些后端服务器的存在，而以为所有资源都来自愿这个反向代理服务器。

Nginx安装与配置

1.安装
brew install nginx
2.配置
sudo vim /usr/local/etc/nginx/nginx.conf

3.常用命令
启动nginx sudo nginx
重启nginx sudo nginx -s reload
停止nginx sudo nginx -s stop

Nginx反向代理

1.通过修改hosts文件模拟多个域名访问本机的nginx。
sudo vim /etc/hosts

2.修改nginx配置文件设置反向代理
sudo vim /usr/local/etc/nginx/nginx.conf

3.重启nginx服务
sudo nginx -s reload
此时，在浏览器访问testnginx.com会访问到www.bilibili.com;访问testreverse.com会访问到www.acfun.cn;访问testproxy.com会访问到www.zhihu.com。

MySQL重置密码

Posted on 2017-10-06 | In DataBase

MySQL安装时会有一个临时密码，如下图：

在知道临时密码的情况下，我直接使用
mysqladmin -u root password 'your new password'
然而，今天发现重启过电脑之后就不再生效了。

改用以下方法重置密码：
1.停止MySQL服务器
2.cd /usr/local/mysql/bin
3.sudo su
4.输入开机密码
5.sudo /usr/local/mysql/bin/mysqld_safe --skip-grant-tables
6.重新打开一个新的终端
7.sudo /usr/local/mysql/bin/mysql -u root
8.输入开机密码
9.UPDATE mysql.user SET authentication_string=PASSWORD('your new password' WHERE User='root';
10.FLUSH PRIVILEGES;
11.\q

修改密码后确实能够使用新密码登录，但可能会遇到如下错误：
You must reset your password using ALTER USER statement before executing this statement
解决方法如下：
1.mysql>SET PASSWORD = PASSWORD('your new password');
2.mysql>ALTER USER 'root'@'localhost' PASSWORD EXPIRE NEVER;
3.mysql>FLUSH PRIVILEGES;

MySQL数据库优化

Posted on 2017-10-03 | In DataBase

1.表的设计要规范。

2.适当建立索引，在频繁作为检索条件，更新较少的字段上建立索引，以提高查询速度。

3.分表查询，包括水平分割和垂直分割。

4.读写分离，读(read)，写(create、update、delete)。

5.建立存储过程。

数据库范式：1NF、2NF、3NF、BCNF、4NF、5NF（由高到低）。

规范化：一个低一级范式的关系模式通过模式分解可以转换为若干个高一级范式的关系模式的集合。

1NF: 每一个分量都是不可分割的数据项。

2NF: 不存在非主属性对码的部分函数依赖。

3NF: 不存在非主属性对码的传递函数依赖。

BCNF: 决定因素都是码。不存在主属性对码的部分和传递函数依赖。

4NF: 不存在多值依赖。

Redis安装及配置

Posted on 2017-10-03 | In DataBase

Redis简介

Redis是一个高性能的key-value数据库，是开源的并遵守BSD协议的。

Redis的特点：

  • 支持数据的持久化，可以将内存中的数据保存到磁盘中，重启的时候可以再次加载进行使用。
  • 不仅仅支出简单的key-value类型的数据，同时还提供list,set,hash等数据结构的存储。
  • 支持数据的备份，即master-slave模式的数据备份。

Redis的优势：

  • 性能极高。Redis读的速度是110000次/秒，写的速度是81000次/秒。
  • 丰富的数据类型。Redis支持String,List,Hash,Set等数据类型操作。
  • 原子性。Redis所有操作都是原子性的，同时还支持对几个操作合并后的原子性执行。
  • 丰富的特性。Redis还支持publish/subscribe，通知，key过期等等特性。

Redis安装

  1. wget http://download.redis.io/releases/redis-3.2.9.tar.gz
  2. tar xzf redis-3.2.9.tar.gz
  3. mv redis-3.2.9 redis
  4. make
  5. make test
  6. make install

Redis配置

修改配置文件以开启进程守护模式

sudo vim /usr/local/etc/redis.conf

Redis常用命令

关闭命令：

简单理解JVM

Posted on 2017-10-02 | In Java

JVM(Java Virtual Machine),一种能够运行Java字节码的虚拟机，以堆栈结构机器来进行实做。

源文件.java经编译器编译成字节码文件，字节码文件.class经解释器解释翻译成为可执行程序。JVM里面包含解释器。JVM屏蔽了与具体操作系统平台相关的信息，使得Java程序只需生成在Java虚拟机上运行的字节码，就可以在多种平台上不加修改的运行。

JVM的特征：

  • JVM是基于栈的虚拟机
    X86和ARM是基于寄存器运行的，而JVM是基于栈运行的。
  • 符号引用
    除原始数据类型以外的所有类型（类和接口）是通过符号引用，而不直接通过内存地址引用。
  • 垃圾回收
    类的实例是通过用户代码手动创建，但是通过垃圾回收自动销毁。

垃圾回收GC(Garbage Collection)，是Java语言中一个显著的特点，使得Java程序员在编写程序的时候不再考虑内存管理。垃圾回收可以有效的防止内存泄露，有效的使用空闲的内存。其中，内存泄露是指该内存空间使用完毕之后未回收。

对GC来说，当对象被创建的时候，GC就开始监控这个对象的地址、大小及使用情况。通常，GC采用有向图的方式记录和管理堆中的所有对象。通过这种方式确定那些对象是“可达的”，哪些对象是“不可达的”。当GC确定一些对象是“不可达”的时候，就可以回收这些内存空间。我们也可以通过手动执行System.gc()来通知GC运行，但是Java语言规范并不保证GC一定会执行。

Java八大排序之快速排序

Posted on 2017-10-01 | In Java

快速排序基本思想：
选择一个基准元素，通常选择第一个元素，通过一趟扫描，将待排序列分成两部分，一部分比基准元素小，一部分大于等于基准元素，此时基准元素在其排好序后的正确位置，然后再用同样的方法递归地排序划分的两部分，直到最后每部分只剩一个元素。

eg.初始状态：15，5，16，2，25，8，20，9，18，12

选择第一个元素15，15在所有元素中排第六，将它归位
第一轮：12，5，9，2，8，15，20，25，18，16

选择每次划分的左右两部分的第一个元素，即12，20
第二轮：8，5，9，2，12，15，16，18，20，25

选择每次划分的左右两部分的第一个元素，即8，16，25
第三轮：2，5，8，9，12，15，16，18，20，25

选择每次划分的左右两部分的第一个元素，即2，9，18
第四轮：2，5，8，9，12，15，16，18，20，25

选择每次划分的左右两部分的第一个元素，即5
第五轮：2，5，8，9，12，15，16，18，20，25

快速排序时间复杂度：
数据随机分布时最好，为O(nlog2n)
数据基本有序时最坏，为O(n^2)
平均情况为O(nlog2n)

快速排序空间复杂度：O(log2n)

快速排序并不产生一个有序区，但每一趟都会归位一个元素，因此快速排序是不稳定的。


Java八大排序之冒泡排序

Posted on 2017-10-01 | In Java

冒泡排序基本思想：
在要排序的一组数中，对当前还未排好序的范围内的全部数，自上而下对相邻两个数依次进行比较和调整，让比较大的数往下沉，较小的数往上冒。即：每当相邻的数比较后发现它们的排序与排序要求相反时，就将它们互换。

eg. 初始状态 23 36 28 16
第一轮： 23 36 16 28
第二轮： 23 16 36 28
第三轮： 16 23 36 28
第四轮： 16 23 28 36

冒泡排序时间复杂度：
正序最好为O(n)
反序最坏为O(n^2)
平均情况为O(n^2)

冒泡排序空间复杂度：O(1)

冒泡排序产生的有序区一定是全局有序的，每一趟排序都会归为一个元素，所以冒泡排序是稳定的。
