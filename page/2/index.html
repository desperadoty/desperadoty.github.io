4NF: 不存在多值依赖。

Redis安装及配置

Posted on 2017-10-03 | In DataBase

Redis简介

Redis是一个高性能的key-value数据库，是开源的并遵守BSD协议的。

Redis的特点：

  • 支持数据的持久化，可以将内存中的数据保存到磁盘中，重启的时候可以再次加载进行使用。
  • 不仅仅支出简单的key-value类型的数据，同时还提供list,set,hash等数据结构的存储。
  • 支持数据的备份，即master-slave模式的数据备份。

Redis的优势：

  • 性能极高。Redis读的速度是110000次/秒，写的速度是81000次/秒。
  • 丰富的数据类型。Redis支持String,List,Hash,Set等数据类型操作。
  • 原子性。Redis所有操作都是原子性的，同时还支持对几个操作合并后的原子性执行。
  • 丰富的特性。Redis还支持publish/subscribe，通知，key过期等等特性。

Redis安装

  1. wget http://download.redis.io/releases/redis-3.2.9.tar.gz
  2. tar xzf redis-3.2.9.tar.gz
  3. mv redis-3.2.9 redis
  4. make
  5. make test
  6. make install

Redis配置

修改配置文件以开启进程守护模式

sudo vim /usr/local/etc/redis.conf

Redis常用命令

redis-server redis服务器
redis-cli redis客户端
redis-benchmark redis性能测试工具
redis-check-aof AOF文件修复工具
redis-check-rdb RDB文件修复工具

检查是否启动:
redis-cli ping

关闭命令：
redis-cli shutdown

简单理解JVM

Posted on 2017-10-02 | In Java

JVM(Java Virtual Machine),一种能够运行Java字节码的虚拟机，以堆栈结构机器来进行实做。

源文件.java经编译器编译成字节码文件，字节码文件.class经解释器解释翻译成为可执行程序。JVM里面包含解释器。JVM屏蔽了与具体操作系统平台相关的信息，使得Java程序只需生成在Java虚拟机上运行的字节码，就可以在多种平台上不加修改的运行。

JVM的特征：

  • JVM是基于栈的虚拟机
    X86和ARM是基于寄存器运行的，而JVM是基于栈运行的。
  • 符号引用
    除原始数据类型以外的所有类型（类和接口）是通过符号引用，而不直接通过内存地址引用。
  • 垃圾回收
    类的实例是通过用户代码手动创建，但是通过垃圾回收自动销毁。

垃圾回收GC(Garbage Collection)，是Java语言中一个显著的特点，使得Java程序员在编写程序的时候不再考虑内存管理。垃圾回收可以有效的防止内存泄露，有效的使用空闲的内存。其中，内存泄露是指该内存空间使用完毕之后未回收。

对GC来说，当对象被创建的时候，GC就开始监控这个对象的地址、大小及使用情况。通常，GC采用有向图的方式记录和管理堆中的所有对象。通过这种方式确定那些对象是“可达的”，哪些对象是“不可达的”。当GC确定一些对象是“不可达”的时候，就可以回收这些内存空间。我们也可以通过手动执行System.gc()来通知GC运行，但是Java语言规范并不保证GC一定会执行。

Java八大排序之快速排序

Posted on 2017-10-01 | In Java

快速排序基本思想：
选择一个基准元素，通常选择第一个元素，通过一趟扫描，将待排序列分成两部分，一部分比基准元素小，一部分大于等于基准元素，此时基准元素在其排好序后的正确位置，然后再用同样的方法递归地排序划分的两部分，直到最后每部分只剩一个元素。

eg.初始状态：15，5，16，2，25，8，20，9，18，12

选择第一个元素15，15在所有元素中排第六，将它归位
第一轮：12，5，9，2，8，15，20，25，18，16

选择每次划分的左右两部分的第一个元素，即12，20
第二轮：8，5，9，2，12，15，16，18，20，25

选择每次划分的左右两部分的第一个元素，即8，16，25
第三轮：2，5，8，9，12，15，16，18，20，25

选择每次划分的左右两部分的第一个元素，即2，9，18
第四轮：2，5，8，9，12，15，16，18，20，25

选择每次划分的左右两部分的第一个元素，即5
第五轮：2，5，8，9，12，15，16，18，20，25

快速排序时间复杂度：
数据随机分布时最好，为O(nlog2n)
数据基本有序时最坏，为O(n^2)
平均情况为O(nlog2n)

快速排序空间复杂度：O(log2n)

快速排序并不产生一个有序区，但每一趟都会归位一个元素，因此快速排序是不稳定的。


Java八大排序之冒泡排序

Posted on 2017-10-01 | In Java

冒泡排序基本思想：
在要排序的一组数中，对当前还未排好序的范围内的全部数，自上而下对相邻两个数依次进行比较和调整，让比较大的数往下沉，较小的数往上冒。即：每当相邻的数比较后发现它们的排序与排序要求相反时，就将它们互换。

eg. 初始状态 23 36 28 16
第一轮： 23 36 16 28
第二轮： 23 16 36 28
第三轮： 16 23 36 28
第四轮： 16 23 28 36

冒泡排序时间复杂度：
正序最好为O(n)
反序最坏为O(n^2)
平均情况为O(n^2)

冒泡排序空间复杂度：O(1)

冒泡排序产生的有序区一定是全局有序的，每一趟排序都会归为一个元素，所以冒泡排序是稳定的。

TCP三次握手及四次挥手

Posted on 2017-09-30

OSI七层模型：
物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。

TCP/IP协议族常用协议：
应用层：HTTP,FTP,DNS
传输层：TCP,UDP
网络层：IP,ICMP
数据链路层：PPP

建立TCP连接的三次握手过程：

  1. Client给Server发送一个SYN报文请求
  2. Server给Client发送SYN+ACK报文，告知对方已收到请求
  3. Client再给Server一个ACK报文回应

中断连接端可以是Client也可以是Server

断开TCP连接的四次挥手过程：

  1. Client给Server发送FIN报文请求
  2. Server给Client发送ACK报文，告知对方已收到请求
  3. Server给Client发送FIN报文
  4. Client给Server发送ACK报文回应

为什么断开连接需要四次挥手呢？
原因是在断开连接的时候，Server端收到FIN报文，可能并不会立即关闭Socket，而是先回复一个ACK报文，告诉Client自己已收到请求，等Server端的所有报文都发送完，才会发送FIN报文给Client。这时候，Client收到FIN报文就知道可以断开连接了，就给Server发送一个ACK报文回应。

Java多线程

在OS中引入进程的目的是为了使多个程序能并发执行，以提高资源利用率和系统吞吐量，引入线程的目的是为了减少程序在并发执行时所付出的时空开销，使OS具有更好的并发性。

在引入线程的OS中，进程是拥有资源的基本单位，线程是调度和分派的基本单位。

Java集合框架

Posted on 2017-09-28 | In Java

集合类型主要分为Set(集), List(列表), Map(映射)。
集合类存放的是对象的引用，而非对象本身。

List

List接口继承Collection接口，允许集合中存在重复项。List中的元素是有序的。

常用的实现类：

  • ArrayList
    可变大小的数组。
  • LinkedList
    是一种可以在任何位置进行高效的插入和删除操作的有序序列。

Set

Set接口继承Collection接口，不允许集合中存在重复项。Set中的元素是无序的。

常用的实现类：

  • HashSet
    按照哈希算法存取集合中的对象，存取速度比较快。允许包含值为null的元素，但最多只能有一个null元素。采用散列的存储方法，所以没有顺序，元素的插入顺序和输出顺序不一致。
  • TreeSet
    实现了SortedSet接口，能够对集合中的对象进行排序。
  • LinkedHashSet
    保存了元素的添加顺序，所以其中元素是有序的。

Map

Map接口是一组成对的键-值对象，即所持有的是key-value pairs。Map中不允许有重复的key。

常用的实现类：

  • HashMap
    根据key的HashCode值存储数据，根据key可以直接获取它的value，具有很快的访问速度。最多只允许一条记录的key为null(多条会覆盖)，允许多条记录的value为null。非同步。
  • TreeMap
    其中键值对默认按照key升序排序。使用Iterator遍历时得到的结果是有序的。不允许key的值为null。非同步。
  • LinkedHashMap
    保存了记录的添加顺序，使用Iterator遍历时，先得到的记录肯定是先添加的。允许key和value为null。非同步。
  • Hashtable
    不允许key和value的值为null。同步。

HTTP状态码及含义

Posted on 2017-09-28 | In NetWork

HTTP状态码(HTTP Status Code)是用以表示网页服务器(Web Server)超文本传输协议(HTTP)响应状态的3位数字代码。

1xx 消息，服务器成功收到请求，需要请求者继续执行操作

2xx 成功，操作被成功接收并处理

3xx 重定向，需要进一步的操作以完成请求

4xx 客户端错误，请求包含语法错误或无法完成请求

5xx 服务器错误，服务器在处理请求的过程中发生了错误

常见HTTP状态码如下：

200 OK 请求成功

301 Moved Permanently 资源被永久转移到其他URL

400 Bad Request 客户端请求的语法错误，服务器不能或不会处理该请求

401 Unauthorized 当前请求需要用户验证

403 Forbidden 服务器已经理解请求，但还是拒绝执行它

404 Not Found 请求失败，服务器无法根据客户端的请求找到资源

500 Internal Server Error 内部服务器错误

502 Bad Gateway 充当网关或代理的服务器，从远端服务器接收到了一个无效的请求

504 Gateway Time-out 充当网关或者代理的服务器，未及时从远端服务器获取请求
