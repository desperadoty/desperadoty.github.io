Jenkins Publish HTML & Performance 插件配置(二）

Posted on 2019-06-16 | In Jenkins

每次构建都会生成html和jtl文件，随着构建次数的增加，文件就会越来越多。之前想的过于简单，在Ant构建之前执行rm -rf命令，现在发现这样会导致Performance Trend展示异常。

更新思路：每次执行构建之前，执行cd $path && rm `ls -t |tail -n +2` ，只保留上一次构建的结果，其余删除。

另外，之前未发现使用jmeter-results-shanhe-me.xsl时html报告未获取到请求具体信息，比如：请求方式、请求参数及响应数据。

更新配置：修改jmeter.properties
