Android APP性能测试基础

发表于 2019-08-18 | 分类于 APP

adb devices

查看CPU占用率的两条命令：

①adb shell top -n 1

②adb shell dumpsys cpuinfo | grep pkgname

基本数据存储位置

adb shell
cd /proc/$pid
ls

查看程序的内存使用情况:

adb shell dumpsys meminfo pkgname

VSS - Virtual Set Size 虚拟耗用内存
RSS - Resident Set Size 实际物理内存
PSS - Proportional Set Size 比例分配物理内存（统计内存大小）
USS - Unique Set Size 进程独占内存（测试内存泄露）

一般来说，以上四个数据依次递减

帧率，最直接的影响是界面的流畅度

adb shell dumpsys gfxinfo pkgname

安卓⼿机屏幕的最⾼高帧率是60帧/秒， 那么为了保持这个帧率，一个界⾯面的渲染时间不不能超过1/60秒,即16.7ms

计算方法：
实时帧率 = 1 / ⼀个界⾯渲染所花费的时间
帧率 = 60-(单位时间内所有问题帧的耗时/16.7)

手机开发者选项开启GPU呈现模式分析

界面重绘

手机开发者选项开启调试GPU过度绘制

浅紫 → 浅绿 → 浅红 → 红色
1x → 2x → 3x → 4x
尽量避免3x、4x重绘超过屏幕的1/3

①adb logcat ActivityManager:D

②录屏后使用工具拆帧，计算耗时

adb shell screenrecord /sdcard/test.mp4
adb pull /sdcard/test.mp4
brew install ffmpeg
ffmpeg -i test.mp4 %d.jpg

④自动化脚本判断元素出现，计算耗时
⑤traceview

电量
清空耗电数据

adb shell dumpsys batterystats --reset
adb shell dumpsys batterystats --enable full-wake-history

获取电量数据

adb shell dumpsys batterystats >battery.txt
adb bugreport bugreport.zip
adb bugreport >bugreport.txt
adb shell bugreport >/sdcard/bugreport.txt

可使用工具Battery Historian 2.0
