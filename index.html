查看程序的内存使用情况:

adb shell dumpsys meminfo pkgname

VSS - Virtual Set Size 虚拟耗用内存
RSS - Resident Set Size 实际物理内存
PSS - Proportional Set Size 比例分配物理内存（统计内存大小）
USS - Unique Set Size 进程独占内存（测试内存泄露）

一般来说，以上四个数据依次递减

帧率，最直接的影响是界面的流畅度

adb shell dumpsys gfxinfo pkgname

安卓⼿机屏幕的最⾼高帧率是60帧/秒， 那么为了保持这个帧率，一个界⾯面的渲染时间不不能超过1/60秒,即16.7ms

计算方法：
实时帧率 = 1 / ⼀个界⾯渲染所花费的时间
帧率 = 60-(单位时间内所有问题帧的耗时/16.7)

Jmeter Plugins - Once Only Controller

Posted on 2019-06-18 | In Jmeter

顾名思义，Once Only Controller在每个线程组仅执行一次。

一般将它作为Loop Controller的子节点，常常应用于用户登录场景。

eg.设置所有请求循环两次，但对登录请求添加Once Only Controller。


执行测试计划并查看结果，可见登录请求仅执行一次，其他请求执行两次。

Jenkins Publish HTML & Performance 插件配置(二）

Posted on 2019-06-16 | In Jenkins

每次构建都会生成html和jtl文件，随着构建次数的增加，文件就会越来越多。之前想的过于简单，在Ant构建之前执行rm -rf命令，现在发现这样会导致Performance Trend展示异常。

更新思路：每次执行构建之前，执行cd $path && rm `ls -t |tail -n +2` ，只保留上一次构建的结果，其余删除。

另外，之前未发现使用jmeter-results-shanhe-me.xsl时html报告未获取到请求具体信息，比如：请求方式、请求参数及响应数据。

更新配置：修改jmeter.properties
